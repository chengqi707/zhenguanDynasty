# 凌烟阁密档 - MVP产品需求文档

## 1. 产品概述

| 项目 | 内容 |
|------|------|
| 产品名称 | 凌烟阁密档 |
| 一句话描述 | 扮演李世民，在凌烟阁功臣的明争暗斗中维持平衡，书写你的贞观传奇 |
| 目标平台 | Web浏览器（PC+手机自适应）→ 后续可扩展微信小程序/Android WebView |
| 技术栈 | HTML5 + CSS3 + 原生JavaScript（零框架依赖，单文件即可运行） |
| MVP周期 | 2周（1周核心玩法，1周打磨+测试） |
| 成功标准 | 玩家能完成一局游戏（约10-15分钟），并愿意重玩尝试不同结局 |

---

## 2. 核心玩法设计

### 2.1 游戏循环（单局10-15分钟）

```
每天上朝 → 处理3个事件 → 晚间决策 → 进入下一天
        ↓
   第15天触发"凌烟阁评定" → 根据关系值判定结局
```

### 2.2 阶段系统（V1.2新增）

15天游戏时间划分为3个历史阶段，每个阶段有独立的事件池和主题：

| 阶段 | 天数 | 历史时期 | 主题 |
|------|------|---------|------|
| **贞观初年** | 1-5 | 627-629 | 巩固皇位、制度改革、整饬吏治 |
| **贞观中期** | 6-10 | 630-639 | 对外战争、盛世气象、太子初立 |
| **贞观晚年** | 11-15 | 640-649 | 立储之争、魏征去世、后宫暗流 |

界面顶部显示当前阶段和对应历史年份（如"贞观初年 · 627年"），增强历史代入感。

### 2.3 核心机制：四势力平衡系统

V1.2版本设计**4个核心人物**，代表四股势力：

| 人物 | 势力 | 性格标签 | 核心诉求 |
|------|------|----------|----------|
| **长孙无忌** | 外戚/文官 | 谨慎、顾全大局 | 希望太子李承乾顺利继位，反对激进改革 |
| **尉迟敬德** | 武将/功臣 | 粗犷、居功自傲 | 要求更多赏赐和地位，不满文官掌权 |
| **魏征** | 谏臣/清流 | 直率、不畏权贵 | 推行新政、抑制功臣特权，哪怕得罪人 |
| **房玄龄** | 谋臣/务实 | 沉稳、善于调和 | 稳定朝政、推进制度改革，调和文武矛盾 |

**关键数值**：
- 每个人物有 **好感度（-100~+100）**
- 每方势力有 **影响力（0~100）**
- 玩家有 **皇权稳定度（0~100）**——任何一方影响力超过80或低于20都会降低稳定度

**胜负条件**：
- **胜利**：15天后皇权稳定度≥60，且至少两方好感度≥0
- **失败**：皇权稳定度降至0，或任意一方好感度降至-100（对方造反/罢朝）

### 2.4 历史人物生老病死规则（V1.2.3新增）

游戏涉及关键历史人物的生死节点时必须**遵守史实**，避免审核风险。当前实现：

| 人物 | 历史节点 | 游戏表现 | 状态变化 |
|------|---------|---------|----------|
| **魏征** | 643年去世（对应late阶段"魏征去世"事件） | 触发该事件后，魏征进入`deceased`状态 | 角色卡片灰显、显示🕯️「故」标签；不可召见；事件效果跳过已故人物 |

**通用规则**：
1. 已故人物不再出现在「单独召见」列表
2. 已故人物卡片以灰色呈现，头像置灰，不可交互
3. 事件效果中若涉及已故人物，自动跳过，避免对不存在的角色进行操作
4. 存档中保存 `deceased` 对象，`loadGame()` 时兼容旧存档（自动补全空对象）
5. 未来新增其他历史人物死亡事件时，统一复用该机制

### 2.5 评价与培养系统扩展草案（待实现）

当前核心指标仅包含「皇权稳定度」与四势力「好感度/影响力」，评价维度相对单一。为提升策略深度和重玩价值，考虑从两个方向扩展：

| 方向 | 思路 | 优点 | 风险 |
|------|------|------|------|
| **A. 新增行政资源指标** | 引入民心、国库、军力等宏观指标 | 更像治国模拟，决策更有层次感 | 指标过多会让单局复杂、单文件代码膨胀、平衡难度大 |
| **B. 大臣属性与培养** | 给每位大臣增加专长属性点（如军事、谋略、外交、政治），通过事件/召见培养，直接影响挑战结果 | 与现有阶段挑战自然结合，给玩家长期培养动力 | 需要重新设计挑战计分公式和事件效果 |

**推荐方案：轻量版双轨扩展**

为控制复杂度，两个方向合并但做轻量处理：

1. **新增 2 个宏观指标**：
   - **民心（0~100）**：反映百姓对朝廷的支持度，影响结局评价和稳定度恢复
   - **国库（0~100）**：反映财政状况，举办宴会、军事行动、大型工程会消耗；合理的经济政策可补充

2. **给每位大臣增加 1 个专长属性**：
   - 专长领域固定，但**能力值（0~100）可成长**
   - 专长与现有势力定位对应：

   | 人物 | 专长领域 | 影响挑战类型 |
   |------|---------|-------------|
   | 尉迟敬德 | 军事 | 决战突厥等军事类决策 |
   | 房玄龄 | 谋略 | 储位暗涌、凌烟阁终评等谋略类决策 |
   | 魏征 | 外交/直谏 | 外交、终评类决策 |
   | 长孙无忌 | 政治 | 朝局平衡、政治斗争类决策 |

3. **培养机制**：
   - **单独召见**时增加二选一：
     - 「勉励」：好感度+15（原效果）
     - 「栽培」：该大臣对应能力值+10，但好感度只+5
   - **事件选项**中部分选项会额外提升/降低大臣能力值
   - 能力值上限 100，下限 0

4. **挑战结果重新挂钩**：
   - 「决战突厥」军事类选项得分 = 尉迟军事能力 × 0.6 + 玩家好感/影响力 × 0.2 + 国库 × 0.2
   - 「储位暗涌」谋略/政治类选项得分 = 房玄龄/长孙能力 × 0.6 + 民心 × 0.2 + 稳定度 × 0.2
   - 「凌烟阁终评」综合得分加入四方平均能力与民心、国库加权

5. **对现有系统的影响**：
   - 事件配置需要新增 `effects.ability` 字段
   - `gameState` 增加 `resources: { people: 50, treasury: 50 }` 和 `characters[key].ability`
   - 挑战配置 `scoreStep` 需要读取新的资源和能力字段
   - 结局评价加入民心、国库、平均能力等维度

**下一步决策**：
- 若确认实现，建议作为 **V1.2.4** 版本，优先实现「大臣能力 + 挑战挂钩」
- 「民心/国库」可作为 V1.2.4 的简化版先接入，后续再扩展更多资源类型
- 实现前需确认：是否接受单 HTML 文件继续膨胀；是否调整现有数值平衡

## 3. 每日游戏流程（详细设计）

### 3.1 上朝阶段（每天自动触发）

**界面布局**：
- 顶部：当前天数（1/15）、皇权稳定度进度条、**当前阶段与历史年份**、设置/档案入口
- 中部：四个人物头像+好感度条（桌面端4列，手机端2x2）
- 下方：事件卡片区域
- 底部：选择按钮

### 3.2 事件系统（V1.2设计45个事件，每天随机抽取3个）

事件库按三个阶段组织：
- **贞观初年（15个）**：玄武门余波、制度改革、科举取士、释放宫女等
- **贞观中期（17个）**：突厥战争、天可汗、吐蕃和亲、高昌远征、太子初立等
- **贞观晚年（13个）**：太子谋反、魏征去世、储位废立、后宫暗流、征高丽等

每个事件包含：
- **stage**：所属阶段（early / middle / late）
- **storyline**：所属剧情线（可选）
- **prerequisite**：前置事件ID（可选）
- **historicalYear**：对应历史年份
- **背景描述**（100字以内，带历史梗）
- **2-3个选项**
- **各选项对不同人物的影响**（好感度±10/20/30）

**剧情线设计（V1.2新增）**：

| 剧情线 | 阶段分布 | 事件链 | 历史依据 |
|--------|---------|--------|---------|
| **制度改革线** | 早期 | 租庸调法 → 均田制 → 修订唐律 → 科举扩招 | 贞观律令格式 |
| **突厥战争线** | 中期 | 突厥犯边 → 李靖出征 → 灭东突厥 → 天可汗尊号 | 630年李靖灭东突厥 |
| **太子储位线** | 中晚期 | 立李承乾 → 太子失德 → 魏王争宠 → 太子谋反被废 → 立李治 | 643年太子承乾谋反 |
| **魏征直谏线** | 全阶段 | 直谏修宫 → 弹劾皇亲 → 临终上疏 → 魏征去世 | 魏征643年去世 |
| **后宫暗线** | 晚期 | 长孙皇后病重 → 长孙皇后去世 → 武则天入宫 → 后宫干政阴影 | 636年长孙皇后去世 |

**事件抽取规则**：
1. 每天从当前阶段事件池中抽取
2. 若某剧情线的前置事件已触发，则优先触发该线后续事件
3. 当前阶段未触发事件不足时，允许已触发事件重复出现，保证每局不卡死
4. 关键剧情节点（如"太子谋反"）必须完整触发前置链后才会出现

**示例事件：突厥边报紧急**

> 突厥骑兵犯边，边关告急。尉迟敬德请战，长孙无忌主张和谈，魏征认为应先整饬内政。

| 选项 | 效果 |
|------|------|
| A. 派尉迟敬德出征 | 尉迟+25，长孙-10，魏征-5，稳定+5 |
| B. 派使者议和 | 长孙+20，尉迟-25，魏征+10，稳定-5 |
| C. 召房玄龄商议后发兵 | 尉迟+10，房玄龄+15，长孙-5，魏征-5，稳定+5 |

---

### 3.3 阶段过渡挑战（V1.2.3新增）

为增强阶段切换的历史仪式感与策略深度，在每个阶段过渡（含结局前）插入一个**短挑战/小游戏**。挑战以弹窗形式覆盖主界面，必须完成才能进入下一阶段/结局。

| 挑战 | 触发时机 | 主题 | 玩法 | 评分依据 |
|------|---------|------|------|----------|
| **决战突厥** | 第6天进入贞观中期 | 630年李靖灭东突厥 | 连续3道战略决策 | 选择策略与对应人物的好感/影响力挂钩 |
| **储位暗涌** | 第11天进入贞观晚年 | 太子承乾与魏王李泰之争 | 连续3道制衡决策 | 选择对朝局平衡的贡献值 |
| **凌烟阁终评** | 第15天结局前 | 评定二十年功过 | 连续3道价值取舍 | 选择贡献值 + 最终人物好感均值 |

**通用规则**：
- 每道决策有4个选项，对应不同势力或价值取向
- 根据累计得分评定结果等级，并影响稳定度与人物数值
- 挑战结果记入本局编年史，用于结局评语
- 挑战过程中不自动保存，完成后统一 `saveGame()`
- 存档中保存 `completedChallenges`（是否完成）和 `challengeResults`（结果等级）

#### 3.3.1 决战突厥

**背景**：贞观三年，东突厥颉利可汗率十万铁骑南下，长安震动。满朝文武意见纷纭，你需在三道关键决策中把握战机。

| 轮次 | 问题 | 选项 | 策略类型 | 计分说明 |
|------|------|------|---------|----------|
| 1 | 突厥先锋逼近泾阳，长安震动。先命何人应对？ | 派尉迟敬德领兵迎击 | 军事 | 基于尉迟敬德 `(好感+影响力)/2` + 随机浮动 |
| | | 令房玄龄统筹粮草、调度后勤 | 后勤 | 基于房玄龄 `(好感+影响力)/2` + 随机浮动 |
| | | 遣魏征出使，以仁义拖延敌军 | 外交 | 基于魏征 `(好感+影响力)/2` + 随机浮动 |
| | | 召长孙无忌稳定朝局、筹措内应 | 权术 | 基于长孙无忌 `(好感+影响力)/2` + 随机浮动 |
| 2 | 李靖上书请求率三千精骑夜袭定襄，是否准许？ | 准，出奇制胜 | 军事 | 同上 |
| | | 先保障粮草辎重，再议出兵 | 后勤 | 同上 |
| | | 发檄文先声夺人，动摇突厥军心 | 外交 | 同上 |
| | | 暂缓，待朝议统一后再定 | 权术 | 同上 |
| 3 | 颉利可汗遣使请和，实则欲图喘息。如何应对？ | 尉迟敬德追击，不给喘息 | 军事 | 同上 |
| | | 房玄龄接管降卒、安顿后勤 | 后勤 | 同上 |
| | | 魏征主张受降，彰显天朝气度 | 外交 | 同上 |
| | | 长孙无忌建议分化突厥部众 | 权术 | 同上 |

**结果判定**：

| 累计得分 | 结果 | 效果 |
|----------|------|------|
| ≥110 | 大捷 | 稳定度+15，尉迟好感+10/影响力+10，房玄龄好感+5，魏征好感+5，长孙好感+5 |
| ≥80 | 小胜 | 稳定度+8，尉迟好感+5/影响力+5，房玄龄好感+3 |
| ≥50 | 僵持 | 稳定度-5，尉迟好感-5 |
| <50 | 溃败 | 稳定度-15，尉迟好感-10/影响力-10，房玄龄好感-5 |

#### 3.3.2 储位暗涌

**背景**：太子李承乾与魏王李泰各树朋党，朝局暗流涌动。你的每一步权衡，都将决定贞观继承人的命运。

| 轮次 | 问题 | 选项 | 平衡贡献 |
|------|------|------|----------|
| 1 | 太子近来骄纵，东宫属官多有不法。你决定： | 派魏征教导太子 | +15 |
| | | 命房玄龄调查东宫属官 | +10 |
| | | 私下敲打太子，削减用度 | +10 |
| | | 召长孙无忌共商储事 | +5 |
| 2 | 魏王李泰献《括地志》，朝野称颂，隐有夺嫡之势。你决定： | 采纳魏征谏止，外王不宜干政 | +20 |
| | | 令房玄龄校书，收归中枢 | +15 |
| | | 厚赏魏王，助长其声望 | +5 |
| | | 交长孙无忌处置 | +5 |
| 3 | 太子承乾谋反迹象渐露，是否易储？ | 听魏征劝谏，暂缓废立 | +20 |
| | | 命房玄龄暗中查证 | +15 |
| | | 命尉迟敬德加强宫禁 | +10 |
| | | 立即废太子，立晋王李治 | -15 |

**结果判定**：

| 累计平衡值 | 结果 | 效果 |
|------------|------|------|
| ≥45 | 朝局稳定 | 稳定度+10，四方好感+5 |
| ≥25 | 平稳过渡 | 稳定度+5 |
| <25 | 储位动荡 | 稳定度-10，四方好感-5 |

#### 3.3.3 凌烟阁终评

**背景**：贞观之治已近尾声，史官请你对二十年功过做最后评定。这将载入凌烟阁，流传后世。

| 轮次 | 问题 | 选项 | 历史遗产贡献 |
|------|------|------|--------------|
| 1 | 凌烟阁功臣排序，你首推何人？ | 论功行赏，不分文武 | +20 |
| | | 首功给魏征等谏臣，以正朝风 | +15 |
| | | 首功给房玄龄等谋臣，以彰治世 | +10 |
| | | 首功给尉迟敬德等武将，以酬军功 | +5 |
| 2 | 如何看待一生的直谏之臣？ | 虚心纳谏，制度化言路 | +20 |
| | | 褒奖魏征，树为楷模 | +15 |
| | | 适可而止，维护皇权威严 | +10 |
| | | 厌恶逆耳之言，后人莫学 | -10 |
| 3 | 你将如何安排身后之事？ | 遗诏明确储君，不委政大臣 | +20 |
| | | 托孤房玄龄，以谋臣辅政 | +10 |
| | | 托孤长孙无忌，以国舅维稳 | +5 |
| | | 未做明确安排，顺其自然 | -5 |

**最终得分** = 选择贡献值 + `round(四方平均好感 / 5)`

| 最终得分 | 结果 | 效果 |
|----------|------|------|
| ≥70 | 贞观遗风 | 稳定度+15 |
| ≥45 | 治世华章 | 稳定度+8 |
| <45 | 功过留待后人评 | 稳定度-5 |

### 3.4 晚间决策（每天事件处理完后）

玩家可以选择一项**晚间行动**（每天限1次）：

| 行动 | 效果 | 限制 |
|------|------|------|
| 单独召见某人 | 该人物好感度+15，但另外两人好感度-5（觉得被冷落） | 每人每3天只能召见1次 |
| 举办宴会 | 所有人好感度+5，但消耗10点皇权稳定（国库支出） | 皇权稳定≥20时可用 |
| 批阅奏折（跳过） | 无变化，进入下一天 | 无限制 |

## 4. 结局系统（MVP设计5个结局）

第15天根据最终数值判定：

| 结局名称 | 触发条件 | 描述 |
|----------|----------|------|
| **贞观之治** | 稳定度≥80，三方好感度均≥20 | 你成功平衡了各方，开创了盛世 |
| **君臣相得** | 稳定度≥60，至少两方好感度≥0 | 虽有波折，但总体平稳 |
| **功高震主** | 尉迟影响力≥80 | 尉迟敬德势力过大，你不得不杯酒释兵权 |
| **外戚专权** | 长孙影响力≥80 | 长孙无忌成为实际掌权者，你逐渐沦为傀儡 |
| **权相治国** | 房玄龄影响力≥80 | 房玄龄权倾朝野，你成为垂拱而治的象征 |
| **众叛亲离** | 稳定度≤0或任意好感度-100 | 政变爆发，你的统治结束了 |

### 4.2 综合评价与谥号系统

第15天进入"凌烟阁评定"时，除了判定结局，系统会根据整局表现给出：

- **综合评价**：一段50-100字的总结评语
- **谥号**：如"太宗""武帝""炀"等，作为玩家成就标签

**谥号判定规则（V1.1设计7个谥号）**：

| 谥号 | 触发条件 | 历史含义 |
|------|---------|---------|
| **太宗** | 稳定度≥80，四方好感度均≥20 | 开创盛世的贤君 |
| **武帝** | 尉迟影响力≥80，且稳定度≥40 | 武功赫赫但朝堂暗流涌动 |
| **文宗** | 魏征影响力≥80，且稳定度≥40 | 善于纳谏、重视文治 |
| **高宗** | 长孙影响力≥80，且稳定度≥40 | 外戚势大，皇权旁落 |
| **宣宗** | 房玄龄影响力≥80，且稳定度≥40 | 知人善任，制度严明 |
| **炀** | 稳定度≤0，或任意好感度-100 | 众叛亲离、统治失败 |
| **中宗** | 不满足以上任何条件 | 守成之君，无功无过 |

**综合评价维度**：

| 维度 | 计算方式 | 评价倾向 |
|------|---------|---------|
| 朝堂平衡 | 四方好感度标准差 | 差值越小，评价越偏向"善于制衡" |
| 皇权稳固 | 最终稳定度 | 越高越偏向"乾纲独断" |
| 势力倾向 | 影响力最高的一方 | 决定评价中突出哪一方 |
| 关键事件 | 玩家在某些特殊事件中的选择 | 加入1-2句个性化评语 |

**示例评语**：

> 你维持了朝堂的基本平衡，长孙无忌与魏征各得其所。虽然尉迟敬德偶有怨言，但大局仍在掌控之中。后世史官称你为**中宗**。

> 你倚重魏征推行新政，朝堂风气为之一新，但武将集团渐生不满。你的统治以文治见长，谥号**文宗**。

**新增系统**：
- **阶段系统**：`config.stages` 划分三阶段，`randomEvents()` 按当前天数过滤
- **剧情链系统**：事件 `prerequisite` 字段 + `triggeredEvents` 记录，实现连贯叙事
- **历史档案系统**：人物生平、隐藏故事、真实历史时间线、本局编年史
- **存档系统**：`localStorage` 自动保存当前进度，支持刷新后继续
- **档案解锁**：结局时根据数值解锁隐藏故事，持久化到 `lyg_archive_unlocks`
- **埋点系统**：`logEvent()` 将关键行为写入 `localStorage`，支持导出 JSON 分析
- **去广告标记**：`noAds` 通过 `localStorage` 持久化，购买后隐藏所有广告按钮
- **音效系统**：Web Audio API 生成简短提示音，无需外部音频文件
- **分享卡片**：`html2canvas` 将结局页生成图片分享
- **人物生死状态（V1.2.3）**：`deceased` 记录已故人物，事件效果和召见菜单自动过滤
- **阶段过渡挑战（V1.2.3）**：`config.stageChallenges` + `gameState.completedChallenges/challengeResults` 实现阶段小游戏
- **版本号检测（V1.2.3）**：`config.version` 与 `localStorage` 中 `lyg_version` 对比，版本变化时自动清理旧存档，避免配置不兼容

**兼容性处理**：
- `loadGame()` 读取旧存档时，自动补全 `triggeredEvents`、`deceased`、`completedChallenges`、`challengeResults` 等新增字段
- 版本号变化时，`checkVersion()` 清除 `lyg_save`、`lyg_logs`、`lyg_archive_unlocks`，确保玩家从新版本配置开始
- 清空旧存档后，新一局会正确进入阶段过渡挑战流程

### 5.3 关键交互逻辑

```
每天流程：
1. 检查是否进入新阶段/结局：若当前天是某阶段 startDay 且该阶段挑战未完成，先弹出阶段过渡挑战
2. 挑战完成后，生成当天3个事件
3. 随机抽取3个未出现过的事件（或允许重复，更简单）
4. 显示事件描述和选项
5. 玩家选择后，更新数值，显示反馈动画
6. 3个事件处理完后，进入晚间决策
7. 晚间决策后，day++，检查是否触发结局条件
8. 如果 day > 15，先进入最终挑战「凌烟阁终评」，再进入最终评定
```

### 5.4 视觉风格建议

- **色调**：暗金+朱红+墨黑，营造朝堂庄重感
- **UI风格**：扁平化+中国风元素（边框用回纹，按钮用印章样式）
- **动画**：数值变化时用Tween动画（可用简单CSS transition），选择后有"朱批"盖章效果
- **字体**：标题用书法体（Google Fonts: Noto Serif SC），正文用宋体

### 5.5 移动端适配要求（V1.2+ 重点）

游戏核心使用场景为**安卓手机微信/浏览器直接打开 HTML 文件**，因此移动端体验必须优先保障：

#### 5.5.1 一屏显示原则

- **禁止整页滚动**：所有核心信息（顶部状态、角色面板、事件/选项或晚间决策）必须在首屏内完整展示
- **允许局部滚动**：事件描述、晚间决策卡片、结局页等模块内部可滚动，但页面主体不出现整体滚动条
- **实现方式**：通过 `flex` 布局 + `overflow: hidden` 控制，`game-area` 占据剩余空间，内部区域按需滚动

#### 5.5.2 安卓适配要点

| 要素 | 要求 |
|------|------|
| 角色面板 | 手机端 2×2 紧凑布局，头像、名字、好感/影响力条均缩小字号和间距 |
| 事件卡片 | 标题、描述、选项在同一区域内；描述区固定高度，超出时内部滚动 |
| 选项按钮 | 减小 padding 和字号，保证 3 个选项 + 反馈信息不溢出 |
| 晚间决策 | 2 列紧凑卡片，必要时内部滚动 |
| 全屏按钮 | 顶部提供全屏切换按钮，调用 Fullscreen API，提升沉浸感 |
| 弹窗 | 弹窗高度不超过 80vh，内容过多时内部滚动 |

#### 5.5.3 兼容性与性能

- 不依赖外部图片资源，所有视觉元素用 CSS/Emoji 实现
- 微信内置浏览器兼容性：避免使用过于前卫的 CSS 特性，Fullscreen API 采用 try-catch 兜底
- 触屏友好：按钮尺寸不小于 44×44px 的可点击区域

### 7.2 V1.0.5 + V1.1 商业化与内容落地方案

已接入以下功能：

| 商业化点 | 形式 | 触发位置 | 实现方式 |
|----------|------|----------|----------|
| **自定义埋点** | 记录玩家行为 | 全局 | `logEvent()` 写入控制台 + localStorage，支持导出 JSON |
| **激励广告** | 看广告获得奖励 | 晚间决策"看广告召见心腹" | 模拟广告占坑，统计点击率 |
| **读档重开** | 看广告或付费 | 结局失败/结束后 | 激励广告位，统计意愿 |
| **去广告付费** | 一次性付费 | 结局页 | ¥6占位按钮，localStorage标记`noAds=true` |
| **结局分享** | 生成图片 | 结局评定页 | `html2canvas`生成结局卡片图 |

**商业化原则**：
1. Web版本不真收款、不接真实广告SDK，只用占位按钮和埋点验证玩家意愿
2. 广告只出现在自然断点（一天结束、一局结束、玩家主动选择奖励）
3. 广告奖励为"额外一次晚间行动"，不直接修改好感度
4. 真正变现等V1.4微信小程序版本再接入微信支付的完整闭环

## 8. 给你的特别建议

1. **文案是你的护城河**：这游戏的乐趣核心在于"历史梗"和"人物性格"。你已经在荣耀工作4年，对职场政治应该深有体会——**把现代职场的人际关系映射到历史人物上**，会让文案更有共鸣。比如魏征就像那个"总是怼你的耿直同事"，尉迟敬德像"有功劳就飘的老员工"。

2. **数值平衡不要纠结**：MVP阶段不用追求绝对平衡，先让10个朋友试玩，根据反馈调整。事件影响的数值可以先放大（±30），让变化更明显，玩家更有感知。

3. **先写事件再写代码**：用Excel或Notion先把10个事件的设计填好（描述、选项、效果），再开始写代码。这样写代码时就是"填空"，不会边写边想导致卡顿。

4. **第一天就部署**：哪怕只有一个空白页面，也先部署到GitHub Pages。给自己一个"已经上线了"的心理暗示，动力会完全不一样。

5. **商业化不要前置到MVP**：MVP的核心目标是验证"玩家愿不愿意玩完一局"。商业化按钮只做占位和埋点，不真收钱、不接真实广告SDK。等留存数据出来再决定V1.1要不要做小程序版本。

6. **谥号是很好的传播点**：玩家看完结局后，给自己一个"太宗"或"炀"的标签，天然适合截图分享。结局页要设计得足够有仪式感，这是免费的社交传播来源。
